Run the cycle once, end to end, and you will drive a blank operation all the way to a closed one. You work across two surfaces. The web app is where you capture intel and review what your agent did, and your coding agent is where the work happens. The `naholo` CLI sits underneath both, and you install it once.

## Prerequisites and setup

Lay the gear once per machine.

- A naholo account and a project, because operations belong to projects.
- macOS. Linux and Windows support is coming later.
- A local repository for the agent to ship code into.
- Node.js v22 or newer, with either npm or pnpm.
- [Claude Code](https://claude.com/claude-code) as your coding agent. Codex support is coming later.
  - If you use [VS Code](https://code.visualstudio.com/), the official Claude Code extension is highly recommended.
  - The Claude desktop app is not recommended, because the cycle requires reviewing a local file, `OPERATION.md`.

Install the CLI, sign in, and wire the repository to a project.

```bash
npm install -g @naholo/cli
naholo login

# team setup: writes the project wiring into the repo so teammates inherit it
naholo init
# solo setup: keeps the same wiring under ~/.naholo/ and commits nothing
naholo covert init
```

## 1. Create an operation and `/infil`

In the web app, open a new operation, title it after the problem, and drop logs for every pain and constraint. Then run:

```
/infil <operationNumber>
```

Or skip the web app and run this in your agent:

```
/fob <title>
<first log (optional)>
```

`/fob` creates the operation server-side and chains `/infil` in one step, landing you in the same place.

## 2. `/warno`

```
/warno
```

It writes the WARNING ORDER with the architecture decisions. Read the Constraints and fix the direction before you move on.

## 3. `/opord`

```
/opord
```

It cuts the WARNING ORDER into single-commit-sized tasks in the OPERATION ORDER and mirrors them into `TASKS.md`. Skim the task list.

## 4. `/splash`

```
/splash
```

It ships the next task, writes the task's After-Action Report, and stops. Review the report and the diff, then run `/splash` again, and repeat until every box in `TASKS.md` is checked. (Run `/opord` if you need to change the upcoming tasks.)

## 5. `/exfil`

```
/exfil
```

It pushes all notes and agent stats to the web app. If every task is finished, it also closes the operation.

> **Tip:** Run `/clear` before `/infil`ing a new operation to start the next one with a clean context window.

[Workflow](/field-manual/workflow) covers every branch and side flow, including the `/recon` route that talks the idea out against the codebase before it becomes an operation.
